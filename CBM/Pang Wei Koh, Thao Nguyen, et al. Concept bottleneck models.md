# Pang Wei Koh, Thao Nguyen, et al. Concept bottleneck models

[https://arxiv.org/abs/2007.04612](https://arxiv.org/abs/2007.04612)

### 模型结构

```python
Image x （Input Layer）

↓ 经过网络g

Concept c （Concept Layer）

has red head -> 1
has blue wing -> 1
has black tail -> 1

↓ 经过网络f

Bird Species y （Output Layer）
```

- 训练集为$\{(x^{(i)}, c^{(i)}, y^{(i)}\}$, 其中$c \in \mathcal{R}^k$, k是概念数目

- 模型所有的信息，必须经过Concept Layer，没有旁路

### 2种Accuracy

#### Task Accuarcy：

预测值 $\hat{y} = f(g(x))$ 与类别标签一致的比率

#### Concept Accuarcy：

概念预测值 $\hat{c} = g(x)$ 与概念标签(Concept Label)一致的比率

### 3种训练方式 + 1对照 + 1对照

|方法|训练方式|优点|缺点|
| ----------------------| -----------------------------------------------------------------------------------------| ---------------------------------------------| ----------------------------------------------------------|
|**Independent Bottleneck**|分别训练 g: ($x \rightarrow c$) 和 f: ($c \rightarrow y$)，这里的f使用真实标签c训练|Concept 学得最准确、模块独立|训练使用真实 Concept、测试使用预测 Concept，存在分布偏移|
|**Sequential Bottleneck**|先训练 ($x \rightarrow c$)，再用预测 Concept 训练 ($c \rightarrow y$)|训练与测试一致，更符合真实部署|第一阶段 Concept 错误会传递给第二阶段|
|**Joint Bottleneck**|整个网络联合优化，Loss \= Task Loss + λ × Concept Loss|分类性能通常最好，可共同优化 Concept 和任务|需要调节 λ；Concept 可能为提高任务性能而偏离真实语义|
|Standard(对照组)|网络结构与bottleneck网络一致(有概念层)，但是损失函数不考虑概念损失项，直接使用Task Loss|||
|NOBOTTLENECK(对照组)|没有中间概念层，backbone直接接一层FC到200类|||

> 测试时**Independent Bottleneck**使用预测的标签推理

### 训练配置和细节

实验采用了两个数据集 CUB 和 OAI，对应分类和回归模型，检测它们的task error（衡量**类别预测**准确程度）和concept error（衡量**概念预测**准确程度）。CUB和OAI均包含人工标注的概念标签和类别标签。

**CUB数据集**：200类鸟种分类，**分类**问题。n = 11788

- **概念**：k\=112个二元鸟类属性（翅膀颜色、喙形状等），因原始标注噪声大，用"多数投票"在类别级别去噪

**模型**：

**概念预测**g：x -> c    使用**Inception V3**（在ImageNet上预训练，除了全连接层），concept损失采用BCELoss

**类别预测**f:    c -> y 使用1层全连接，y损失采用CrossEntropyLoss

**预处理**：训练时随机颜色抖动、随机水平翻转、随机裁剪至299x299分辨率，推理时中心裁剪后resize到299×299

**训练方法：表1**

**超参数搜索**: 在验证集上搜索

- 学习率范围：[0.001, 0.01]
- 学习率调度：恒定不变，或每 [10,15,20] 个epoch衰减10倍直到降到1e-4
- 正则化强度：[0.0004, 0.00004]
- 找到最优超参数后，在train+validation合并数据集上重新训练直至收敛
- Batch size \= 64
- 对于**Joint Bottleneck训练方法，搜索超参数**​$\lambda$使得验证集上在保持较高的concept准确率的同时，最大化task accuarcy
- 优化器：SGD，momentum \= 0.9
- 对concept loss函数，每个概念对总概念损失的贡献权重相等，但每个概念的二元交叉熵损失会按该概念的类别不平衡比例（平均约1:9）加权归一化

---

**OAI数据集**：从膝关节X光片预测 KLG（4级骨关节炎严重程度评分），**回归**问题。n = 36369

- **概念**：k\=10个由放射科医生标注的临床概念（关节间隙变窄、骨刺、钙化等）

**模型**：

**概念预测** ResNet-18（ImageNet预训练，微调最后12个卷积层。最后接一个**单层全连接层**，回归到概念 c；**MSE计算概念损失**

**分数(KLG)预测**：一个3层MLP负责 c→y 输出单一的KLG标量预测值(**MSE计算Task损失**)

**训练方法：表1**

- Batch size \= 8
- 数据增强：随机水平/垂直平移
- 优化器：​**Adam**​（β₁\=0.9, β₂\=0.999）
- 初始学习率：网格搜索于 [0.00005, 0.0005, 0.005]
- 学习率调度：每10个epoch衰减2倍
- 对于**Joint Bottleneck训练方法，搜索超参数**​$\lambda$使得验证集上在保持较高的concept准确率的同时，最大化task accuarcy
- 训练30个epoch，配合早停
- 最终权重取验证集上KLG的RMSE最低的那个epoch

### Benchmark：任务精度与概念精度对比

两个数据集分别是分类(CUB)和回归(OAI)：CUB 报 **error(越低越好)** ，OAI 报 **RMSE(越低越好)** ，均在**留出测试集**上、多随机种子的 ±2SD。

**Table 1｜任务误差 task error**

|模型|CUB y error|OAI y RMSE|
| ---------------------| --------------| --------------|
|Independent|0.240±0.012|0.435±0.024|
|Sequential|0.243±0.006|0.418±0.004|
|Joint|0.199±0.006|0.418±0.004|
|Standard(对照)|0.175±0.008|0.441±0.006|
|No bottleneck(对照)|0.173±0.017|0.443±0.008|
|Multitask(对照)|0.162±0.002|0.425±0.010|

**Table 2｜概念误差 concept error**

|模型|CUB c error|OAI c RMSE|
| ------------------------| --------------| --------------|
|Independent|0.034±0.002|0.529±0.004|
|Sequential|0.034±0.002|0.527±0.004|
|Joint|0.031±0.000|0.543±0.014|
|Standard[linear probe]|0.093±0.004|0.680±0.038|
|SENN[probe]|—|0.676±0.026|

- **任务精度可打平黑箱**：OAI 上 joint/sequential 的 RMSE 甚至优于 standard；CUB 上 sequential/independent 的 0-1 error 略差于 standard，但 joint(允许调整概念以提升任务)补回大部分差距。去掉瓶颈层的 no-bottleneck 与 standard 几乎无差，说明瓶颈结构本身不损失任务精度。
- **概念精度高、且与任务精度不冲突**：OAI 上所有瓶颈模型每个概念的 Pearson 相关 ≥0.87；Figure 2-左在这两个任务上看不到"任务 vs 概念精度"的权衡前沿。
- **数据高效**：Figure 2-右，OAI 上 sequential 只用 ≈25% 训练数据即追平用全量数据的 standard。

### Benchmarking post-hoc concept analysis

概念瓶颈模型能从瓶颈层直接**读出**概念。对照问题：普通黑箱模型能否事后(post-hoc)用**线性探针 linear probe**从中间特征里"读出"这些概念？若能，就不必显式训练概念。

作者按 Kim et al.(2018) 的做法，在 standard 模型每层特征上训练线性探针预测概念(Table 2 底部)：

- OAI：最好探针的概念 RMSE **0.68**，从瓶颈层直接读只有 **0.53**；平均 Pearson 从 0.84 掉到 0.72。
- CUB：探针概念 error **0.09** vs 瓶颈 **0.03**；平均 F1 从 0.92 掉到 0.77。
- 自解释网络 SENN 也得到更低的概念精度。

事先知道要用哪些概念时，直接拿它们训练比训练完再从黑箱恢复要准得多。而且探针即便能读出概念也**无法据此干预**——没法通过改探针输出去改变黑箱决策，这正是下一节的前提。

### 测试时干预 Test-time Intervention —— 本文最核心的创新

黑箱模型没有、而 CBM 独有的能力，也是全文重点。

**做法**：预测显式经过概念层 $\hat c = g(x)$，测试时用 oracle(模拟领域专家，如放射科医生)查询某概念真值 $c_j$，把预测的 $\hat c_j$ **替换**成 $c_j$，让改动沿 $f$ 传播到最终预测 $\hat y$。Figure 3 给了"改一个概念就纠正错误预测"的例子。

- **OAI(回归)** ：直接用真值替换 $\hat c_j$，按验证集算出的**输入无关固定顺序**依次干预。只查询 **2 个概念**，任务 RMSE 就从 >0.4 降到 ≈0.3(Figure 4-左)。
- **CUB(分类)** ：概念是 logits 不能直接拷贝，改成把 $\hat\ell_j$ 设到训练分布的**第5/第95百分位**(真值0设5、1设95)使 $\sigma(\hat\ell_j)$ 逼近真值；相关概念**分组**一起干预，只干预图中**可见**的概念。

**关键发现：不同训练方式的可干预性差别很大，且这个差别看不出自任务/概念精度。**

- **Independent 最适合被干预**：全部概念替换成真值时它误差最低(尽管不干预时略差)。因为它的 $f$ 就是用**真实 0/1 概念**训练的，测试替换回真值正好回到训练分布；sequential/joint 的 $f$ 用**预测概念** $\hat c$ 训练，分布不一致。
- **两个反直觉消融**：① joint 的 λ 太小(0.01，概念没对齐好)时，用真值替换反而**增大**误差(Figure 4-左 "control")；② 把 $c\to y$ 从 3 层 MLP 换成**线性**层，干预效果明显变差(Figure 4-中)——干预前任务/概念精度都相近，作者也说不清原因，留作 future work。
- CUB 上另试 "Joint, from sigmoid"($f$ 接概率 $\sigma(\ell)$ 而非 logits)：正常任务误差更高(0.224 vs 0.199)，但干预更有效、更直接。

任务精度和概念精度**不足以**判断一个模型好不好干预——模型的归纳偏置决定了它如何处理干预前后 $\hat c\to c$ 的分布变化。作者据此推测：单个放射科医生 + 瓶颈模型协作，可能胜过医生或模型单独工作。

### 背景偏移下的鲁棒性 TravelingBirds

作者造了 **TravelingBirds**：CUB 变体，训练集里每个鸟类被**人为绑定一种固定背景**(Places 场景)，测试时把"类别↔背景"映射**打乱**——制造一个只在训练集成立的**伪相关**。

**Table 3｜背景偏移下的误差**

|模型|y error|c error|
| -------------| --------------| --------------|
|Standard|0.627±0.013|—|
|Joint|0.482±0.018|0.069±0.002|
|Sequential|0.496±0.009|0.072±0.002|
|Independent|0.482±0.008|0.072±0.002|

瓶颈模型任务误差**显著低于** standard(0.48 vs 0.63)。因为概念**跨类别共享**(很多鸟都有"黑翅膀")，训练集里就出现在各种背景下，与背景相关性弱；standard 直接 $x\to y$ 会利用"背景=类别"的捷径，映射一变就崩。

> 作者的诚实说明：瓶颈是否更鲁棒**取决于概念的选择和偏移类型**，并非任何一组概念都必然更鲁棒。

### 理论分析（附录C）

良态线性回归设定下(输入 $X$ 正态、概念 $C=XB+\epsilon_1$、目标 $Y=Cb+\epsilon_2$)，independent 瓶颈相对 standard 的**超额误差比**满足 $\le \frac{\frac{k}{d}\sigma_Y^2+\sigma_C^2}{\sigma_Y^2+\sigma_C^2}$：当**概念数** **$k$** **远小于输入维度** **$d$**、且**概念噪声远小于目标噪声**时，瓶颈模型反而更优，为"加一个信息瓶颈却不掉精度"提供了理论直觉。

### 创新点与在 CBM 谱系中的定位

**本文贡献**

1. 一个把任意 DNN 变成 CBM 的极简方案：把某一层 resize 成 $k$ 个神经元对齐 $k$ 个概念，加一个逐分量的概念对齐损失即可，**测试时不需要概念标注**。
2. 系统比较 independent / sequential / joint 三种训练方式 + standard / no-bottleneck / multitask 对照。
3. 证明瓶颈模型**任务精度可打平黑箱、同时概念精度高**，两者在其任务上不冲突。
4. **测试时干预**这一黑箱没有的能力，并揭示不同训练方式的干预权衡。
5. 概念**直接训练优于事后探针恢复**。
6. 对**背景伪相关更鲁棒**、且**更数据高效**。

**对应到《概念瓶颈模型》综述里的贡献坐标** —— 本文(综述里的 [5])是几乎每条演进轴的**原点/基线**，后续工作都在放宽它的某个假设：

|综述中的位置|本文对应做法|后续如何放宽|
| ---------------------------------------------| -----------------------------------------------------| ---------------------------------------------------|
|概念监督 → 人工概念监督(Early CBMs)[5]|用人工标注概念(CUB 属性 / OAI 临床发现)强监督概念层|词典 / CLIP / LLM 引导，走向 label-free、开放词汇|
|瓶颈设计 → ① Hard Bottleneck(原始 CBM)|所有信息必须过离散概念层、无旁路|soft / hybrid / 残差 / 概率 / 能量瓶颈|
|概念相关性建模 → indep. / joint|三种训练方式即"独立 vs 联合"的原型|图 / 层级 / 因果 / 自回归建模概念间关系|
|概念干预 → Independent Intervention|假设概念独立，直接替换单个概念值|自回归 / 随机 / 能量传播(Concept Propagation)|
|概念定位 → 人工选择|人工或验证集固定顺序决定改哪个概念|主动学习、不确定性排序自动定位|
|概念交互 → 传统 CBM(Original Intervention)|用户直接改概念数值 / logits|自然语言(Chat-CBM)、空间区域(SALF)|
|模型适应 → 静态 CBM(Early CBMs)|概念集固定，更新需重训|Editable / TTA / 部署后局部修正|

这篇 2020 年的工作确立了 CBM 的标准范式(预测概念 → 用概念预测标签)与**可干预**这一核心卖点，是整张综述分类体系的共同祖先。
